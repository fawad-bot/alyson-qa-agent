Triggers & roles

How a run starts on its own, and who hears about it. This is what turns on-demand QA into autonomous-on-deploy QA.

Triggers — the deploy webhook
Post-deploy webhook Not connected

When connected, app.alyson.ai fires an event the moment a page deploys and a run is enqueued automatically — no human in the loop. Requires: a signed endpoint, pipeline access to register the hook, an HMAC secret, the trigger contract below, and an idempotency key.

Trigger contract (deploy-event payload)

event:        "page.deployed"
page_url:     "https://…/<slug>"
environment:  "staging" | "production"   → selects tier (prod: T0/T1 · staging: up to T2)
template_id:  "value-estimator"          → fan-out + owner fallback
deploy_id:    "rel_… / commit SHA"       → idempotency + audit
owner:        { id, email }              → drives notification routing
timestamp:    ISO-8601
signature:    HMAC over the body         → authenticity

Status: demand-only today. The contract is the single missing piece for "QA starts itself." Auto-fix + routing still need repo/PR, vault, and alerting grants. Production deploy stays human-gated by design.
